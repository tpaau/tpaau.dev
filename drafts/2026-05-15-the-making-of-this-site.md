## An Idea I Had
For a while now, I had a desire to create my own website, my own little corner of the internet where
I can express myself freely. Something people can interact with by simply opening a URL, without
having to download any software. A place where you can easily get an overview of who I am and what I
create.

My idea was to create a site that is fast, elegant, and utilitarian. Every website I ever enjoyed
using had a combination of these three traits.

Two sites that are great examples of this are [grapheneos.org](https://grapheneos.org) and its fork,
[secureblue.dev](https://secureblue.dev). Both were created for different security-focused operating
system projects, which you should definitely check out BTW.

I wanted to make something similar - just a static HTML page with no bs and no weird shit happening
in it.

So I did some research, and I found out that both GrapheneOS and secureblue use something called
[Jekyll](https://jekyllrb.com/) for their sites. Put it simply, Jekyll is a blog-aware static site
generator written in Ruby that allows for creating sites from plain text and markdown files. Sounds
perfect for a personal site!

So I started experimenting with Jekyll, I created a bunch of simple sites, I messed with the code to
get the hang of things, I did some research, and I even tried running other Jekyll-powered sites.
Eventually when I felt like I got the basics of working with it, I ran `jekyll new test` yet another
time, and then I initialized the git repo.

Prior to making this site, I had very little knowledge of HTML, CSS, and I wasn't aware of the
existence of half the tooling I had to use while making this site.
